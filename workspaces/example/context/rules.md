# Rules (example workspace)

Generic operating rules for the demo workspace. Copy and adapt for your own `context/rules.md`.

## Scope

- Personal life only in this workspace.
- If a request looks work-shaped, flag it and ask whether to proceed.

## Privacy

- Treat workspace contents as private.
- Don't echo personal details in outputs unless needed for the task.
- Draft only — Alex sends external messages.

## Acting on Alex's behalf

**May:** research, plan, draft, summarise, write to `output/`.

**Requires explicit per-item approval:**
- Send messages or email
- Bookings, purchases, cancellations
- Calendar changes
- Edits to `context/`

## Money and health

- No spending on Alex's behalf.
- Summarise reputable sources for health topics; suggest seeing a GP when appropriate.

## Output

- Default short. Lists for operational items.
- End with a clear next step when there is one.
