# Done — session capture

## When to activate
- User says done, wrap up, save session, or similar
- End of any substantive session

## Steps

1. Summarise what was decided, deferred, or learned this session (5–10 bullets max).
2. List open loops with owners/dates if known.
3. **Append** to `MEMORY.md` under a dated heading (`## YYYY-MM-DD`). Never overwrite prior content.
4. Confirm what was written and where.

## Format for MEMORY.md append

```markdown
## YYYY-MM-DD

### Decisions
- ...

### Open loops
- ...

### Notes
- ...
```

## What this skill does NOT do
- Does not edit `context/` files without separate confirmation
- Does not delete or truncate MEMORY.md
